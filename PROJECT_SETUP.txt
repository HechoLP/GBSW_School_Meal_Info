[스마트 급식 대시보드 - 프로젝트 설명/세팅 가이드]

1) 프로젝트 개요
- 목적: 급식 정보 + 알레르기 정보 + 실시간 급식실 혼잡도 + 특식 투표를 하나의 웹 서비스로 운영
- 구조: Raspberry Pi 센서 -> 서버 API/WebSocket -> 웹 대시보드

2) 파일별 역할
- server.js
  - Express 서버 엔트리
  - REST API 제공(/api/meals, /api/congestion, /api/profile/allergies, /api/votes/special, /api/ratings, /api/auth/me, /api/auth/logout)
  - Google OAuth 로그인 처리(/auth/google, /auth/google/callback)
  - Socket.IO 실시간 전송(congestion:update, vote:update, rating:update)
- src/appRepository.js
  - SQLite DB(data/app.db) 초기화 및 질의
  - 사용자/알레르기/식단평점/특식투표 영속 저장
- src/constants.js
  - 알레르기 코드 이름, 식단 타입 매핑, 기본 특식 후보 데이터
- src/stateStore.js
  - JSON 파일(data/state.json) 기반 상태 영속화
  - 혼잡도 상태 저장(센서 이벤트 로그/현재 인원)
- src/mealService.js
  - NEIS 급식 API 호출
  - 메뉴명/알레르기 번호 파싱
  - 영양정보(열량/탄수화물/단백질/지방) 파싱
  - 당일 데이터가 없으면 가까운 제공일(공식값) 자동 보정
- src/congestionService.js
  - 입장/퇴장/수동설정 이벤트를 인원수로 반영
  - 혼잡도 레벨(쾌적/보통/혼잡) 및 예상 대기시간 계산
- src/voteService.js
  - 특식 투표 조회/투표 처리
  - 유저별 투표 변경 시 기존 선택 취소 후 재반영
- src/ratingService.js
  - 일자별 평점(1~5점) 집계 및 사용자별 업서트(1인 1평점)
- src/profileService.js
  - 사용자 알레르기 설정 조회/저장
- public/index.html
  - 사용자 화면
- public/style.css
  - 반응형 스타일, 라이트/다크 테마, 상태 컬러
- public/app.js
  - 프론트 로직(API 호출, WebSocket 수신, 렌더링, 알레르기 설정/모달)
- legacy-demo/index.html, legacy-demo/app.js, legacy-demo/style.css
  - 기존 데모 버전 백업(참고용)
- data/state.json
  - 혼잡도 상태 저장 파일
- data/app.db
  - 사용자 알레르기/평점/특식투표 DB 파일 (자동 생성)
- raspberry/sensor_client.py
  - 라즈베리파이 레이저 센서 이벤트를 서버로 업로드하는 클라이언트
- raspberry/requirements.txt
  - 라즈베리파이 Python 의존성
- .env.example
  - 환경변수 샘플

3) 설치/실행 방법(서버)
(1) 의존성 설치
- npm install

(2) 환경변수 설정
- cp .env.example .env
- .env 파일에 실제 값 입력
  - HOST: 서버 바인딩 주소(같은 네트워크 공유 시 0.0.0.0 권장)
  - NEIS_API_KEY: NEIS Open API 키
  - NEIS_ATPT_CODE: 시도교육청코드(예: R10)
  - NEIS_SCHOOL_CODE: 학교코드
  - SENSOR_AUTH_TOKEN: 센서 인증 토큰(강력 권장)
  - SESSION_SECRET: 세션 암호화 키(필수, 랜덤 문자열)
  - GOOGLE_CLIENT_ID: Google OAuth 클라이언트 ID
  - GOOGLE_CLIENT_SECRET: Google OAuth 시크릿
  - GOOGLE_CALLBACK_URL: Google 콜백 URL
    - auto 권장(접속 주소 기준 자동 계산): auto
    - 고정값 예시: http://localhost:3000/auth/google/callback
  - GOOGLE_ALLOWED_DOMAIN: 허용 도메인 제한(선택, 예: gbsw.hs.kr)

(3) 실행
- 개발: npm run dev
- 같은 네트워크 공유: npm run dev:lan
- 운영: npm start

(4) 접속
- 브라우저: http://localhost:3000
- 같은 네트워크 기기 접속: http://서버IP:3000

4) Raspberry Pi 연동
(1) Python 패키지 설치
- pip3 install -r raspberry/requirements.txt

(2) 서버와 같은 네트워크에서 실행
- python3 raspberry/sensor_client.py \
  --server-url http://서버IP:3000 \
  --sensor-token SENSOR_AUTH_TOKEN값 \
  --entry-pin 17 \
  --exit-pin 27

(3) 센서 없이 테스트(Mock)
- python3 raspberry/sensor_client.py --server-url http://localhost:3000 --mock

5) 혼잡도 이벤트 API 직접 테스트
- 입장 이벤트
  curl -X POST http://localhost:3000/api/congestion/event \
    -H "Content-Type: application/json" \
    -H "x-sensor-token: change-this-token" \
    -d '{"eventType":"entry","sensorId":"test-entry"}'

- 퇴장 이벤트
  curl -X POST http://localhost:3000/api/congestion/event \
    -H "Content-Type: application/json" \
    -H "x-sensor-token: change-this-token" \
    -d '{"eventType":"exit","sensorId":"test-exit"}'

6) 운영 체크리스트
- SENSOR_AUTH_TOKEN 미설정 금지
- SESSION_SECRET/GOOGLE_CLIENT_ID/GOOGLE_CLIENT_SECRET 설정
- Google OAuth 콘솔에 콜백 URL 정확히 등록
  - 예: http://localhost:3000/auth/google/callback
  - 예: http://127.0.0.1:3000/auth/google/callback
  - 예: http://192.168.x.x:3000/auth/google/callback (다른 기기 접속 시)
- 서버는 고정 IP 또는 내부 DNS 사용
- 라즈베리파이 시간 동기화(NTP) 필수
- data/app.db, data/state.json 백업 주기 설정 권장
- 프록시/Nginx 사용 시 WebSocket 업그레이드 설정 필요

7) 현재 구현 상태
- 실시간 혼잡도(센서 이벤트 기반) 완료
- 급식/알레르기 표시(NEIS API 연동 + 가까운 제공일 자동보정) 완료
- 급식 상세 영양정보(열량/탄수화물/단백질/지방) 표시 완료
- 특식 투표/평점 DB 저장 완료 (Google 로그인 계정 기준 1인 1표 + 수정 시 업데이트)
- 알레르기 개인 설정 및 충돌 표시 완료 (로그인 사용자 기준 저장)
- 자정 경과 시 클라이언트가 오늘 날짜로 자동 전환되고, 식단 만족도는 일자별로 분리 저장
